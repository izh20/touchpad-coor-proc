# HP012 项目 HID IIC 数据上报格式

## 概述

本项目用于解析 HID IIC 手指包数据，实现手指轨迹可视化。

---

## 数据包结构

### 基础包格式 (47 bytes)

| 字节偏移 | 名称 | 说明 |
|---------|------|------|
| 0 | Report ID | 报告ID，固定为 0x01 |
| 1 | Y[7:0] | Y坐标低8位 |
| 2 | X[7:0] | X坐标低8位 |
| 3 | Y[9:8] | Y坐标高2位 (bit7:4) |
| 4 | X[9:8] | X坐标高2位 (bit7:4) |
| 5~50 | Reserved | 保留字节 |

### 5指槽位格式

每个指槽占 **8 bytes**，5个槽位依次排列：

| 槽位 | 起始偏移 | 占用字节 |
|------|---------|---------|
| Slot 0 | Byte 3 | 3, 4, 5, 6, 7, 8, 9, 10 |
| Slot 1 | Byte 11 | 11, 12, 13, 14, 15, 16, 17, 18 |
| Slot 2 | Byte 19 | 19, 20, 21, 22, 23, 24, 25, 26 |
| Slot 3 | Byte 27 | 27, 28, 29, 30, 31, 32, 33, 34 |
| Slot 4 | Byte 35 | 35, 36, 37, 38, 39, 40, 41, 42 |

---

## 单个指槽详细结构

### 槽位内字节定义

| 槽位内偏移 | 名称 | 说明 |
|-----------|------|------|
| 0 | Status / Finger ID | **高4bit**: 手指ID (0-9) |
| 0 | Status | **低4bit**: 状态值 |
| 1 | Pressure | 压力值 |
| 2 | Y[7:0] | Y坐标低8位 (小端) |
| 3 | Y[9:8] | Y坐标高2位 |
| 4 | X[7:0] | X坐标低8位 (小端) |
| 5 | X[9:8] | X坐标高2位 |
| 6 | Reserved | 保留 |
| 7 | Reserved | 保留 |

### 状态定义 (低4bit)

| 状态值 | 含义 | 轨迹行为 |
|--------|------|---------|
| 3 | 手指按下 (Finger Touch) | 显示细线轨迹 + 空心圆 |
| 2 | 大面积按下 (Large Area Touch) | 显示粗线轨迹 + 实心圆 |
| 1 | 手指抬起 (Finger Release) | 清除该手指ID的轨迹 |
| 0 | 大面积抬起 (Large Area Release) | 清除该手指ID的轨迹 |

### 手指ID定义 (高4bit)

| ID值 | 含义 |
|------|------|
| 0 | 手指0 |
| 1 | 手指1 |
| ... | ... |
| 9 | 手指9 |

## 坐标计算公式

```
X = X[15:8] << 8 | X[7:0]  (16bit，小端存储)
Y = Y[15:8] << 8 | Y[7:0]  (16bit，小端存储)
```

---

## 解析示例

### 原始数据示例 (十六进制)

```
01 20 10 30 00 01 00 00 00 00 01 30 20 00 01 00 00 ...
```

### 解析过程

1. **Report ID**: `0x01`
2. **Slot 0 (起始于 Byte 3)**:
   - Byte 3 = `0x30` → 高4bit=`3`(手指ID), 低4bit=`0`(大面积释放)
   - Byte 11 = `0x31` → 高4bit=`3`(手指ID), 低4bit=`1`(手指释放)

---

## 与实际代码的对应关系

根据实际测试数据，每包数据为 **47字节 (0x2F)**，解析关键字节位置：

```python
# 手指槽位起始位置
SLOT_OFFSETS = [3, 11, 19, 27, 35]

# 解析手指ID和状态
finger_id = (byte & 0xF0) >> 4    # 高4bit
status = byte & 0x0F              # 低4bit
```

### 输入文件格式

- 纯文本文件，每行包含一个字节的十六进制值
- 或者连续十六进制字符串
- 每 **47字节** 为一包数据
